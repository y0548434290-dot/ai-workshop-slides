מהקוד שלך — לאוויר

בנית משהו נהדר ב-VS Code. עכשיו נראה איך שומרים אותו, מעלים לענן, ומפרסמים אותו לאינטרנט — צעד אחר צעד, עד אתר חי שאפשר לשלוח לכולם.

אחרי הבנייה
בנית — עכשיו שולחים לעולם
כל מה שבנינו עד עכשיו יושב על המחשב שלך בלבד. כדי שאחרים יראו — צריך "לשלוח" אותו החוצה.
  • Git — שומרים את העבודה כנקודת ציון (commit).
  • GitHub — מעלים את הקוד לענן (push).
  • Vercel — הופכים אותו לאתר חי בכתובת (deploy).
זה בדיוק התהליך שמפתחים אמיתיים עושים כל יום — ואחרי הפעם הראשונה זה לוקח דקה.

GitHub

כל הכלים האלה כבר מותקנים ומחוברים אצלך — ממדריך ההתקנה (מצגת 1).
1Git · שמירה
שומרים את העבודה — Commit
"Commit" = נקודת שמירה עם תיאור קצר. עושים את זה לפני שמעלים. שלוש דרכים:
בתוך VS Codeפאנל Source Control — ויזואלי, מומלץ למתחילות
בטרמינלשתי פקודות קצרות (למטה)
עם Claude Codeפשוט מבקשות ממנו לשמור
git add . git commit -m "your message here"
1Git · בתוך VS Code
לעשות Commit בלחיצות — Source Control
  1. לוחצים על אייקון ה-Source Control בסרגל הצד (או Ctrl+Shift+G).
  2. מעל הקבצים ששונו לוחצים + כדי לכלול אותם בשמירה (Stage).
  3. כותבים הודעה קצרה שמתארת מה שינית — למשל "הוספתי דף צור קשר".
  4. לוחצים על כפתור Commit (✓).
ההודעה עוזרת לך (ול-Claude) להבין בעתיד מה היה בכל שמירה. כדאי לכתוב משהו ברור.
2GitHub · העלאה ראשונה
מעלים את הפרויקט לענן
בפעם הראשונה הכפתור נקרא "Publish to GitHub".
  1. בפאנל Source Control יופיע כפתור Publish Branch / Publish to GitHub.
  2. אם זו הפעם הראשונה — VS Code יבקש להתחבר ל-GitHub (Sign in). מאשרים.
  3. בוחרים שם לריפו, ו-Private (פרטי) או Public (ציבורי).
  4. VS Code יוצר את המאגר ב-GitHub ומעלה אליו את כל הקוד.
Private = רק את רואה את הקוד · Public = כולם. תמיד אפשר לשנות בהמשך בהגדרות הריפו.
2GitHub · השינויים הבאים
כל שינוי הבא: Commit ואז Push
אחרי ההעלאה הראשונה, מעדכנים את הענן עם push.
עורכיםמשנים משהו בקוד
Commitשומרים נקודת ציון
Pushמעלים ל-GitHub
ב-VS Code זה כפתור "Sync Changes" (חצים ↑↓) בפאנל Source Control — הוא דוחף את ה-commits לענן.
3Vercel · התחברות
מתחברים ל-Vercel דרך GitHub
  1. נכנסים לאתר:
vercel.com
  1. לוחצים Sign Up ובוחרים Continue with GitHub.
  2. מאשרים ל-Vercel גישה לחשבון ה-GitHub שלך.
כך Vercel "רואה" את הריפו שהעלית — ויכול למשוך ממנו את הקוד כדי לפרסם אותו.
3Vercel · פרסום
מייבאים את הפרויקט — ומפרסמים
  1. לוחצים Add New ← Project, ובוחרים Import ליד הריפו שלך.
  2. Vercel מזהה אוטומטית את סוג הפרויקט (אין מה להגדיר ברוב המקרים).
  3. לוחצים Deploy ומחכים כמה שניות.
  4. מקבלים כתובת חיה: שם-הפרויקט.vercel.app 🎉
זהו — האתר באוויר! אפשר לשלוח את הקישור לכל אחד, מכל מכשיר.

ובלי קוד?
בנית ב-Base44 או Lovable? — אפילו יותר קל
בכלים האלה אין צורך בשרשרת Git → GitHub → Vercel — הם מארחים את האתר בעצמם.

Base44

מתארח אוטומטית על תשתית Wix. אין שלב "deploy" — האתר חי מהרגע שיצרת אותו.

Lovable

מתארח על Lovable Cloud. לוחצים כפתור Publish — והאתר באוויר.

ההבדל מהשרשרת שלמדנו: כאן הפרסום מובנה — לחיצה אחת. רוצות דומיין משלכן ("לקשר לשרת")? גם זה כמה לחיצות — בשקפים הבאים.
Base44 · פרסום ודומיין
Base44 — פרסום וקישור דומיין משלך

אין שלב פרסום נפרד — האפליקציה חיה מיד, בכתובת חינמית: myapp.base44.app (אפשר לשנות את השם בכל רגע).

  1. נכנסים להגדרות הפרויקט ← Domain.
  2. שתי דרכים: לקנות דומיין דרך Base44 (DNS+SSL אוטומטי, מתחבר תוך 5–30 דק') או לחבר דומיין קיים.
  3. לדומיין קיים: מוסיפים רשומת CNAME אצל ספק הדומיין, עם הערך שמופיע בהגדרות הפרויקט.
ה-SSL (האייקון 🔒) מופק אוטומטית. שינויי DNS עשויים לקחת עד 72 שעות. דומיין מותאם זמין בתוכניות בתשלום.
Lovable · פרסום ודומיין
Lovable — פרסום וקישור דומיין משלך

לוחצים על כפתור Publish (פינה עליונה) ומקבלים כתובת חיה: yourapp.lovable.app — בכל תוכנית, כולל חינם, עם SSL ו-CDN מובנים.

  1. לדומיין משלכן: Project ← Settings ← Domains (דורש תוכנית בתשלום).
  2. רושמים דומיין אצל ספק (Namecheap / GoDaddy).
  3. מעתיקים את ערכי ה-DNS ש-Lovable נותן, ומדביקים אצל ספק הדומיין.
לעדכון האתר אחרי שינויים: Publish ← Update. בונוס: אפשר גם לסנכרן ל-GitHub ולפרסם דרך Vercel/Netlify — בדיוק השרשרת שלמדנו.
