1VS Code · עורך הקוד המרכזי
מה זה VS Code ולמה דווקא הוא?

VS Code (קיצור של Visual Studio Code) הוא עורך קוד — תוכנה שבה כותבים, פותחים ומארגנים קבצי קוד. חשבי עליו כמו "וורד", אבל למתכנתות.

  • קל ומהיר — נטען בשניות, לא מכביד על המחשב.
  • תומך בכל שפות הקוד — צובע ומסדר את הטקסט כך שקל לקרוא.
  • תוספים — אפשר להוסיף יכולות, כולל את Claude Code עצמו (שלב 5).
  • חינמי ומבית Microsoft — הסטנדרט של רוב המפתחים כיום.
ההורדה מהאתר הרשמי בלבד: code.visualstudio.com — חשוב להוריד ממקור רשמי כדי להימנע מגרסאות מזויפות.
1VS Code · התקנה
איך מתקינים — ומה זה "Add to PATH"
  1. נכנסים ל-code.visualstudio.com ולוחצים Download לפי מערכת ההפעלה (Mac / Windows).
  2. מריצים את קובץ ההתקנה ועוקבים אחר ההוראות על המסך.
ב-Windows — בזמן ההתקנה סמני את "Add to PATH".
מה זה PATH? רשימה שבה המחשב מחפש תוכנות כשמקלידים פקודה בטרמינל. כש-VS Code נמצא ב-PATH, אפשר לפתוח אותו מכל מקום בכתיבת code . — בלי זה הטרמינל "לא ימצא" אותו.
2מושג רקע
Git — מכונת הזמן של הקוד
ניהול גרסאות (Version Control): היכולת לשמור צילום מצב של הקוד בכל רגע ולחזור אחורה.
  • כל שינוי נשמר כ-"commit" — נקודת שמירה עם תיאור ותאריך.
  • אפשר לחזור אחורה לכל גרסה קודמת אם משהו נשבר.
  • אפשר לראות מי שינה מה ומתי — חשוב גם בעבודת צוות.
גם Claude Code משתמש ב-Git כדי להבין את מבנה הפרויקט ואת השינויים שנעשו — לכן הוא הכרחי כבר מההתחלה.
2Git · התקנה ובדיקה
התקנה ובדיקה שהכל עובד
  1. נכנסים ל-git-scm.com/downloads ומורידים את הגרסה למערכת שלך.
  2. מריצים את ההתקנה ומשאירים את כל ברירות המחדל (Next, Next, Next).
  3. פותחים טרמינל (שורת פקודה) ומריצים:
git --version
gitהפעלת התוכנה Git.
--version"הצג לי את מספר הגרסה המותקנת". אם מופיע מספר — ההתקנה הצליחה.
2Git · הגדרת זהות
מי "חותם" על הקוד? — git config
לכל commit מצורף שם ומייל. נגדיר אותם פעם אחת לכל המחשב.
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
configפקודת ההגדרות של Git.
--global"לכל הפרויקטים במחשב" — מגדירים פעם אחת ולא בכל פרויקט מחדש.
user.name / user.emailהשם והמייל שיופיעו על כל שמירה. את הערכים במרכאות מחליפים בפרטים שלך.
חשוב מאוד: השתמשי בדיוק באותו מייל שאיתו תפתחי חשבון GitHub (שלב 4) — כך המערכות יזהו שזו את ויסתנכרנו.
3מושג רקע
Node.js — המנוע שמריץ פקודות
סביבת הרצה (Runtime): התוכנה שמאפשרת להריץ כלים שנכתבו בשפת JavaScript מחוץ לדפדפן.
  • ה-CLI של Claude Code רץ על Node.js — בלי Node הוא פשוט לא יפעל.
  • Node מגיע עם npm — מנהל החבילות שמתקין כלים (נשתמש בו בשלב 5).
מה זה LTS? ראשי תיבות של Long-Term Support — גרסה יציבה שמתוחזקת לאורך זמן. תמיד בוחרים אותה ("Recommended for most users") ולא את הגרסה הניסיונית.
דרוש מינימום גרסה 18 ומעלה.
3Node.js · התקנה ובדיקה
התקנה ומספרי גרסה
  1. נכנסים ל-nodejs.org ומורידים את גרסת ה-LTS.
  2. מריצים את ההתקנה לפי ברירות המחדל, ואז בודקים בטרמינל:
node --version
הפלט אמור להתחיל ב-v18 ומעלה, למשל v20.11.0.
v20Major — גרסה ראשית. שינוי כאן עשוי לשבור תאימות.
.11Minor — תוספות ויכולות חדשות, ללא שבירה.
.0Patch — תיקוני באגים קטנים בלבד.
4מושג רקע
GitHub — הענן של המפתחים
נקודה מבלבלת שחשוב לחדד: Git ו-GitHub זה לא אותו דבר.

Git

התוכנה שרצה על המחשב שלך ומנהלת את גרסאות הקוד מקומית.

GitHub

השירות בענן שאליו מעלים את הקוד באמצעות Git — לגיבוי ושיתוף.

למה צריך אותו? כדי לגבות את הקוד, לשתף אותו, ולחבר אותו לשירותי אחסון כך שהאתר או המערכת יהיו באוויר.

4GitHub · פתיחת חשבון
פותחים חשבון — עם אותו מייל
  1. נכנסים ל-github.com ולוחצים Sign up (פינה ימנית עליונה).
  2. מזינים את אותה כתובת מייל שהגדרת ב-Git (שלב 2).
  3. בוחרים סיסמה חזקה ושם משתמש פנוי.
  4. עוקבים אחר הוראות האימות ומאשרים את המייל שמתקבל בתיבה.
שימוש באותו מייל הוא מה שמחבר בין ה-commits במחשב שלך לבין החשבון בענן — כך GitHub יודע שהקוד שייך לך.
5Claude Code · סוכן ה-AI
Claude Code — השותף שכותב קוד

Claude Code הוא סוכן AI מבית Anthropic, שיושב בתוך סביבת העבודה וכותב, מתקן ומסביר קוד יחד איתך — לפי בקשות בשפה רגילה.

  • מבין את כל הפרויקט (בעזרת Git שהתקנו) ולא רק קובץ בודד.
  • יכול להריץ פקודות, לערוך קבצים, ולבדוק שהכל עובד.
זה הכלי היחיד במדריך שדורש מנוי בתשלום. יש לו שתי דרכי התקנה — נראה אותן בשקף הבא.
5Claude Code · שתי דרכי התקנה
מסלול א' (תוסף) או מסלול ב' (טרמינל)

מסלול א' — תוסף מומלץ למתחילות

ב-VS Code: אייקון Extensions או Ctrl+Shift+X ← מחפשים Claude Code (של Anthropic) ← Install. אייקון כוכב יופיע בסרגל.

מסלול ב' — טרמינל (CLI)

פותחים טרמינל ב-VS Code (Ctrl+`) ומריצים את הפקודה למטה.

npm install -g @anthropic-ai/claude-code
npmמנהל החבילות של Node — מתקין כלים מהאינטרנט.
install -g"התקן גלובלית" (global) — זמין מכל מקום במחשב, לא רק בפרויקט אחד.
@anthropic-ai/claude-codeשם החבילה. @anthropic-ai הוא ה-scope (היצרן) ו-claude-code הוא הכלי עצמו.
5Claude Code · מנוי ועברית
המנוי הדרוש ותוסף נוחות לעברית
נדרש מנוי Claude Pro — מינימום $20 לחודש. אפשר להתקין ולהתחבר ישירות מתוך VS Code.
תוסף מומלץ לנוחות בעברית: Claude Code RTL — מייעל את הממשק לכתיבה מימין לשמאל, כך שהטקסט בעברית מיושר נכון.
בסיום שלב זה כל הכלים מותקנים — נשאר רק להתחבר (שלב 6).
6התחברות · השלב האחרון
התחברות ואימות — את כמעט שם!

כדי ש-Claude יוכל לעבוד צריך להתחבר לחשבון. בטרמינל מקלידים פשוט:

claude
  1. ייפתח קישור בדפדפן — מתחברים עם חשבון Claude Pro או עם ה-API Key שלך.
  2. כשהטרמינל מציג Ready — ההתקנה הושלמה ואת מוכנה לפתח! 🎉
Pro מול API Key: מנוי Pro = תשלום חודשי קבוע דרך הדפדפן. API Key = מפתח אישי שמחייב לפי שימוש. למתחילות — מנוי Pro פשוט יותר.
7מושג רקע
Vercel — מעלים את האתר לאוויר
פלטפורמת ענן לפרסום (Deployment) — הופכת את הקוד שכתבת לאתר חי שכל אחד יכול להיכנס אליו.

אחרי שכתבנו קוד ושמרנו אותו ב-GitHub, צריך שירות שיריץ אותו וייתן לו כתובת באינטרנט. זה בדיוק התפקיד של Vercel.

  • מתחברת ל-GitHub — מושכת את הקוד ישירות מהחשבון שלך.
  • בונה ומפרסמת אוטומטית — הופכת את הקוד לאתר פעיל תוך שניות.
  • נותנת כתובת חיה (URL) — לינק לשיתוף שעובד מכל מכשיר בעולם.
חינמי להתחלה — תוכנית ה-Hobby מספיקה לפרויקטים אישיים וללמידה, ללא עלות. ההרשמה ב-vercel.com דרך חשבון ה-GitHub.
7Vercel · איך זה עובד
מהקוד שלך — לאוויר, אוטומטית
  1. כותבות קוד ב-VS Code (בעזרת Claude Code).
  2. שומרות את השינויים עם Git ודוחפות אותם ל-GitHub.
  3. Vercel מזהה את העדכון, מבצע Build (בנייה) ומעלה את האתר.
  4. תוך שניות — האתר באוויר בכתובת קבועה. 🎉
פרסום מתמשך (Continuous Deployment): כל עדכון עתידי שתעלי ל-GitHub יתפרסם לבד — לא צריך להעלות ידנית בכל פעם.
תצוגות מקדימות (Preview): כל שינוי מקבל קישור תצוגה נפרד, כך שאפשר לבדוק לפני שמפרסמים לכולם.
